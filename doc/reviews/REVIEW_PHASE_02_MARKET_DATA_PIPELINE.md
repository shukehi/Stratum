# Review Phase 02: Market Data Pipeline

## 1. 复审目标

确认市场数据摄取层只做事实标准化，不提前掺入策略解释。

## 2. 复审范围

- `src/clients/exchange/*`
- `src/domain/market/*`
- `src/services/market-data/*`
- 对应测试

## 3. 工程复审问题

1. `OHLCV`、`funding`、`OI` 是否被标准化为领域模型。
2. `fetchSpotTicker()` 是否已实现，且在不可用时返回默认值。
3. 时间戳、数值转换和异常处理是否一致。
4. 客户端层是否只负责请求和标准化，而没有策略逻辑。
5. 数据缺失和接口失败是否有清晰行为。

## 4. 第一性原理复审问题

1. 当前数据模型是否保留了后续判断”谁被迫交易”所需的字段。
2. 是否同时具备合约端和现货端的价格数据，以支持基差背离检测。
3. 是否错误地把客户端层变成了解释层。
4. 是否为了图方便省略了 funding、OI 或现货价格，导致后续只能退回图形解释。

## 5. 常见失败模式

- 只接 OHLCV，不接 funding / OI
- 不接现货 ticker，导致基差背离无法计算
- 在客户端层直接推导看多看空
- 时间粒度混乱，无法后续回放

## 6. 通过标准

- 四类核心数据（OHLCV、funding、OI、现货价格）都能稳定获取或优雅降级
- 数据层没有越权承担策略判断
- 数据足以支持后续状态层与参与者层
