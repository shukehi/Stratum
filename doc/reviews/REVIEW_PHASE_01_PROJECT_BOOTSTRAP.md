# Review Phase 01: Project Bootstrap

## 1. 复审目标

确认工程骨架是否稳定、可测试、可扩展，且不会妨碍后续按第一性原理实现策略层。

## 2. 复审范围

- `package.json`
- `tsconfig.json`
- `src/app/*`
- `src/index.ts`
- `test/*`

## 3. 工程复审问题

1. 项目是否能正常安装依赖并运行测试命令。
2. 环境变量读取是否集中且失败行为清晰。
3. 配置是否集中管理，而不是散落在业务文件中。
4. 目录结构是否与主文档一致。
5. 是否存在后续阶段难以接入的硬编码。

## 4. 第一性原理复审问题

1. 当前工程骨架是否为状态层、参与者层和结构层保留了清晰边界。
2. 是否存在一开始就把系统做成“指标脚本”或“单文件脚本”的倾向。
3. 是否有任何结构会诱导后续直接先写 FVG 再补状态层。

## 5. 常见失败模式

- 配置写死在服务代码
- 无测试框架
- 入口文件直接承载业务逻辑
- 缺失日志与错误处理基础设施

## 6. 通过标准

- 工程骨架可运行
- 没有妨碍后续分层的设计
- 不存在明显违背主架构的硬编码
